关于位运算,有一种既熟悉又陌生的感觉.熟悉是因为我们当初学java基础的时候,应该也都学过.并且平常在阅读一些源码的时候经常看见它的身影,比如说HashMap的源码 ↓↓↓

static final int tableSizeFor(int cap) {
        int n = cap - 1;
        n |= n >>> 1;
        n |= n >>> 2;
        n |= n >>> 4;
        n |= n >>> 8;
        n |= n >>> 16;
        return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

陌生是因为日常开发中,大概率是不会去使用它. 结论:你可以不去使用它,但你必须得懂它

关于二进制

一切的起源都要从二进制开始…在了解位之前,有必要先知道二进制的概念

百度百科:二进制

Java 中的二进制

//举个例子:
//int占4个字节（byte）
//1byte = 8bit
//换算下来，一个int类型即占32bit
int i = 5; //这里的5为十进制，转换为二进制为：101，使用完整的32位表示即为：00000000 00000000 00000000 00000101

Java 提供了进制转换的API

public static void main(String[] args) {
    int num = 168;
    //十进制转二进制
    System.out.println(Integer.toBinaryString(num));
    //十进制转八进制
    System.out.println(Integer.toOctalString(num));
    //十进制转十六进制
    System.out.println(Integer.toHexString(num));
    //二进制转十进制
    System.out.println(Integer.valueOf("10101000",2));
    //八进制转十进制
    System.out.println(Integer.valueOf("250",8));
    //十六进制转十进制
    System.out.println(Integer.valueOf("a8",16));
}

输出结果:

10101000
250
a8
168
168
168

在Java7版本以前，Java是不支持直接书写除十进制以外的其它进制字面量。但这在Java7以及以后版本就允许了：

  • 二进制：前置0b/0B
  • 八进制：前置0
  • 十进制：默认的，无需前置
  • 十六进制：前置0x/0X
public static void main(String[] args) {
    //二进制
    int i = 0B101;
    System.out.println(i); //5
    System.out.println(Integer.toBinaryString(i));
    //八进制
    i = 0101;
    System.out.println(i); //65
    System.out.println(Integer.toBinaryString(i));
    //十进制
    i = 101;
    System.out.println(i); //101
    System.out.println(Integer.toBinaryString(i));
    //十六进制
    i = 0x101;
    System.out.println(i); //257
    System.out.println(Integer.toBinaryString(i));
}

输出结果:

5
101
65
1000001
101
1100101
257
100000001

原码 反码 补码

来自百度百科的解释

在计算机内，有符号数有三种表示法：原码、反码以及补码。
 原码：就是二进制定点表示法，即最高位为符号位，“0”正负“1”，其余位表示数值的大小。
 反码：正数的反码与其原码相同；负数的反码是对正数逐位取反，符号位保持为1。
 补码：正数的补码与其原码相同；负数的补码是在其反码的末位加1。

我们已经知道了一个 int 型数值是 4 个字节。每个字节有 8 位。但对于一个 int 或者其它整数类型如 （long）的数值而言还要注意的是，它的最高位是符号位。

  • 最高位为0表示正数。
  • 最高位为1表示负数

原码 将一个数字转换成二进制就是这个数值的原码。

int a =5; //原码  0000 0000 0000 0000 0000 0000 0000 0101
int b =-5;//原码  1000 0000 0000 0000 0000 0000 0000 0101

反码 分两种情况：正数和负数

  • 正数 正数的反码就是原码。
  • 负数 负数的反码是在原码的基础上，符号位不变 其它位都取反。
int a =5; //原码、反码：0000 0000 0000 0000 0000 0000 0000 0101
int a =-5;//原码：1000 0000 0000 0000 0000 0000 0000 0101
int a =-5;//反码：1111 1111 1111 1111 1111 1111 1111 1010

补码 仍然分正数和负数两种情况

  • 正数 正数的补码就是原码。
  • 负数 负数的补码在反码的基础上加1。
int a =5; //原码、反码：0000 0000 0000 0000 0000 0000 0000 0101
int a =-5;//原码：1000 0000 0000 0000 0000 0000 0000 0101
int a =-5;//反码：1111 1111 1111 1111 1111 1111 1111 1010
int a =-5;//补码：1111 1111 1111 1111 1111 1111 1111 1011

在Java中，负数是用补码来表示

public static void main(String[] args) {
    int i = -5;
    System.out.println(Integer.toBinaryString(i));
}

输出结果:

11111111111111111111111111111011

计算机在进行数值运算的时候，是通过补码表示每个数值的。

//比如:5 - 3 = 2
//实际上计算机是这样算的:5+(-3)=2
00000000000000000000000000000101+11111111111111111111111111111011=1 00000000000000000000000000000010

最后的结果是1 0000 0000 0000 0000 0000 0000 0000 0010 这样的二进制，由于 int 类型只有 4 byte，所以最高位产生了溢出，进位 1 被丢弃。结果就变成了 0010 也就是 2。

Java位运算

运算符的运算规则表:

操作符 规则描述
& 如果相对应位都是1，则结果为1，否则为0
I 如果相对应位都是0，则结果为0，否则为1
^ 如果相对应位值相同，则结果为0，否则为1
~ 按位取反运算符翻转操作数的每一位，即0变成1，1变成0
« 按位左移运算符。左操作数按位左移右操作数指定的位数
» 按位右移运算符。左操作数按位右移右操作数指定的位数

&:按位与

规则:如果相对应位都是1，则结果为1，否则为0. (相同得1)

举个栗子:

public static void main(String[] args) {
    int a = 8;//1000
    int b = 9;//1001
    //结果:1000=8
    System.out.println("a的二进制:" + Integer.toBinaryString(a));
    System.out.println("b的二进制:" + Integer.toBinaryString(b));
    System.out.println("二进制结果:" + Integer.toBinaryString(a & b));
    System.out.println("十进制结果:" + (a & b));
}

输出结果:

a的二进制:1000
b的二进制:1001
二进制结果:1000
十进制结果:8

|：按位或

**规则:如果相对应位都是0，则结果为0，否则为1(有1得1) **

举个栗子:

public static void main(String[] args) {
    int a = 8;//1000
    int b = 9;//1001
    //结果:1001=9
    System.out.println("a的二进制:" + Integer.toBinaryString(a));
    System.out.println("b的二进制:" + Integer.toBinaryString(b));
    System.out.println("二进制结果:" + Integer.toBinaryString(a | b));
    System.out.println("十进制结果:" + (a | b));

}

输出结果:

a的二进制:1000
b的二进制:1001
二进制结果:1001
十进制结果:9

^:按位异或

规则:如果相对应位值相同，则结果为0，否则为1(相同得0)

举个栗子:

public static void main(String[] args) {
    int a = 8;//1000
    int b = 9;//1001
    //结果:0001=1
    System.out.println("a的二进制:" + Integer.toBinaryString(a));
    System.out.println("b的二进制:" + Integer.toBinaryString(b));
    System.out.println("二进制结果:" + Integer.toBinaryString(a ^ b));
    System.out.println("十进制结果:" + (a ^ b));
}

输出结果:

a的二进制:1000
b的二进制:1001
二进制结果:1
十进制结果:1

~:按位非

规则:按位取反运算符翻转操作数的每一位，即0变成1，1变成0

举个栗子:

public static void main(String[] args) {
    int a = 8;//1000
    //结果:11111111111111111111111111110111=-9
    System.out.println("二进制结果:" + Integer.toBinaryString(~a));
    System.out.println("十进制结果:" + (~a));
}

输出结果:

a的二进制:1000
b的二进制:1001
二进制结果：11111111111111111111111111110111
十进制结果:-9

«:按位左移

规则:左移n位,右边低位补0

举个栗子:

public static void main(String[] args) {
    int a = 8;//1000
    //左移n位,右边补n个0
    //结果:1000 00 = 32
    System.out.println("a的二进制:" + Integer.toBinaryString(a));
    System.out.println("二进制结果:" + Integer.toBinaryString(a << 2));
    System.out.println("十进制结果:" + (a << 2));

}

输出结果:

a的二进制:1000
二进制结果:100000
十进制结果:32

»:按位右移

规则:正数:右移n位,右边抹掉n位,左边高位补n个0

举个栗子

public static void main(String[] args) {
    int a = 8;//1000
    //正数:右移n位,右边抹掉n位,左边高位补n个0
    //结果:10 = 2
    System.out.println("a的二进制:" + Integer.toBinaryString(a));
    System.out.println("二进制结果:" + Integer.toBinaryString(a >> 2));
    System.out.println("十进制结果:" + (a >>2));

}

输出结果:

a的二进制:1000
二进制结果:10
十进制结果:2

规则:负数:右移n位,右边抹掉n位,左边高位补n个1

举个栗子:

public static void main(String[] args) {
    int a = -8;//11111111111111111111111111111000 负数补码
    //负数:右移n位,右边抹掉n位,左边高位补n个1
    //结果:-2
    System.out.println("a的二进制:" + Integer.toBinaryString(a));//负数补码
    System.out.println("二进制结果:" + Integer.toBinaryString(a >> 2));
    System.out.println("十进制结果:" + (a >>2));

}

输出:

a的二进制:11111111111111111111111111111000
二进制结果:11111111111111111111111111111110
十进制结果:-2

»>:无符号右移

  • 正数:正数右移和无符号右移没有区别
  • 负数:无符号右移n位,左边高位补n个0

举个栗子:

public static void main(String[] args) {
    int a = -5;//11111111111111111111111111111011 负数补码
    //负数:右移n位,右边抹掉n位,符号位补0
    //结果:00111111111111111111111111111110
    System.out.println("a的二进制:" + Integer.toBinaryString(a));//负数是补码
    System.out.println("二进制结果:" + Integer.toBinaryString(a >>> 2));
    System.out.println("十进制结果:" + (a >>> 2));

}

输出结果:

a的二进制:11111111111111111111111111111011
二进制结果:00111111111111111111111111111110
十进制结果:1073741822

总结:有符号、无符号 看的就是二进制的符号位， 无符号:就不管符号位，右移只填充0；有符号，就是符号位是啥，我就填充啥
